## 8. 💎 LIGHTING & DEPTH SYSTEM

### Shadow Hierarchy
```
Micro Shadow:       0 2px 8px rgba(0, 0, 0, 0.12)
Small Shadow:       0 4px 12px rgba(0, 0, 0, 0.15)
Medium Shadow:      0 8px 24px rgba(0, 0, 0, 0.2)
Large Shadow:       0 12px 32px rgba(0, 0, 0, 0.25)
Elevation Shadow:   0 16px 48px rgba(0, 0, 0, 0.3)
```

### Glow Effects
```
Cyan Glow:          0 0 20px rgba(0, 217, 255, 0.3)
Violet Glow:        0 0 20px rgba(176, 38, 255, 0.3)
Blue Glow:          0 0 20px rgba(0, 128, 255, 0.3)
Subtle Glow:        0 0 8px rgba(0, 217, 255, 0.1)
```

### Lighting Rules
- **Key Light**: Electric Blue or Cyan from top-left
- **Fill Light**: Very subtle, same color family, 20% intensity
- **Back Light**: Barely perceptible, creates rim lighting on edges
- **Bloom Effects**: Used sparingly on neon accents only
- **Parallax**: Subtle, 3-5px depth variation for hero sections

## 10. ⚡ MOTION & INTERACTION GUIDELINES

### Animation Timing
```
Micro Interactions:     150ms - 200ms (ease-out)
Standard Transitions:   250ms - 300ms (ease-out)
Complex Animations:     400ms - 600ms (cubic-bezier)
Entrance Effects:       400ms - 500ms (ease-out)
Exit Effects:          250ms - 350ms (ease-in)
```

### Easing Functions
- **Micro**: `cubic-bezier(0.4, 0, 0.2, 1)` (ease-out)
- **Standard**: `cubic-bezier(0.34, 1.56, 0.64, 1)` (ease-out elastic)
- **Smooth**: `cubic-bezier(0.25, 0.46, 0.45, 0.94)` (ease-in-out)

### Motion Rules
- Animations should feel "intelligent" and purposeful
- No bouncy or playful effects
- Subtle parallax creates depth without distraction
- Data visualizations animate with smooth curves
- UI elements move with confidence and clarity
- Hover states provide clear feedback with minimal motion
